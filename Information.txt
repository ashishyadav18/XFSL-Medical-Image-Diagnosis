============================================================
PROJECT SUMMARY
===============

This repository contains my dissertation research project:

"Explainable Prototype-Based Few-Shot Learning for Skin Lesion Classification"

The project combines:

* EfficientNet-B0 feature extraction
* Prototype-Based Few-Shot Learning (FSL)
* Test-Time Augmentation (TTA)
* Power Normalization
* Similarity Gap (Δ) uncertainty estimation
* Out-of-Distribution (OOD) detection
* Grad-CAM explainability

Final deployment version is available separately inside:

deployment_v2/

============================================================
MAIN RESEARCH WORKFLOW
======================

STEP 1
Train CNN Backbone

File:
train_model.py

Purpose:
Trains EfficientNet-B0 on the skin lesion dataset.

Output:
trained_model.pth

This is the primary CNN model used throughout the project.

---

STEP 2
Run Few-Shot Learning Evaluation

File:
main.py

Purpose:
Main dissertation experiment.

Performs:

* N-shot evaluation
* Prototype creation
* TTA feature extraction
* Power normalization
* Similarity-gap analysis

Outputs:

logs/ablation_results.csv
logs/per_class_ablation.csv
logs/k20_detailed_logs.csv
logs/saved_k20_prototypes.npy

This is the most important script in the repository.

---

STEP 3
OOD Detection Evaluation

File:
ood_test.py

Purpose:
Tests Out-of-Distribution rejection capability.

Evaluates:

* Far-OOD samples
* Near-OOD samples

Outputs:

logs/ood_results.csv

Used to select the final threshold:

τ = 0.55

---

STEP 4
Prediction + Explainability

File:
predict_and_explain.py

Purpose:

Runs complete inference pipeline:

Image
→ Feature Extraction
→ Prototype Matching
→ OOD Check
→ Similarity Gap Analysis
→ Grad-CAM

Outputs:

outputs/results.csv
outputs/heatmaps/

Used for qualitative demonstrations.

============================================================
ADDITIONAL EXPERIMENTS
======================

File:
filter_ablation.py

Purpose:

Filtering ablation study.

Tests whether removing support samples far from
the class center improves prototype quality.

Filter Ratios Tested:

0%
5%
10%
15%
20%
25%
30%

Conclusion:

Filtering did not provide meaningful improvement
over the simpler prototype averaging approach.

Therefore this method was NOT used in the final
proposed framework.

Kept for research completeness.

Linear SVM baseline.

baseline_random_forest.py

Random Forest baseline.

baseline_matching_network.py

Matching Network baseline.

baseline_vanilla_protonet.py

Standard ProtoNet baseline.

proposed_fsl.py

Optimized XFSL implementation used only for
baseline comparison runs.

Functionally similar to main.py but without
ablation studies and extensive logging.

Faster execution.

Used when generating comparison tables.
